# Security Policy

## Supported Versions

This project is currently in early development. Security fixes are applied to the latest code on the default branch.

## Reporting a Vulnerability

Please do not open a public issue for sensitive vulnerabilities.

Report security issues privately by contacting the maintainers and include:

- Contact: tobes@longtaildragon.com
- A clear description of the vulnerability.
- Reproduction steps or a proof of concept.
- Impact assessment.
- Suggested mitigation, if known.

## Response Targets

- Initial acknowledgement: within 7 days.
- Triage and severity assessment: as soon as practical after acknowledgement.
- Fix timeline: depends on severity and complexity.

## Disclosure

After a fix is available, maintainers may coordinate a responsible disclosure with the reporter.
